(a) Define the MDP¶

  • $\mathcal{T}$ decision epochs (measured in days)
  • $\mathcal{S}$ states where $s$ represents states; a tuple of number of cars at each location
  • $\mathcal{A}$ actions where each action $a$ can be represented as a tuple that sums to $0$ and

each element represents the number of cars moved in relation to the indexed location. For example $a = \{-1,0,1\}$ would represent the movement of a single car from location 1 (or more properly, location 0) to the third location. It is assumed that one cannot move more cars from a location than exist at that location. So the valid actions are all $a\in\mathcal{A}_s$ for which $a+s$ is non-negative.

First, we import the supporting python libraries and enumerate the constants defined by Sutton and Barto for this problem.

def transition(state,action,
               req_lams=REQUEST_LAM,ret_lams=RETURN_LAM,
               credit=RENT_CREDIT,cost=MOVE_COST,lot_sizes=MAX_CARS):    
    # Customer behavior modeled by Poisson process
    requests = [poisson.rvs(x) for x in req_lams]
    returns  = [poisson.rvs(x) for x in ret_lams]
    # Rentals meet customer requests if the location has enough vehicles
    rentals = np.minimum(state,requests)
    
    # New state is the old state, less rentals
    new_state = np.subtract(state, rentals)
    # plus customer returns
    new_state = np.add(new_state, returns)
    # and transfers
    new_state = np.add(new_state, action)
    # lose excess cars to corporate
    np.minimum(new_state,lot_sizes)
    
    profit = sum(np.multiply(rentals,credit) - np.multiply(action,cost))
    return new_state, profit
  • Transition Probability Function

is a convolution of $\sim\text{poisson}(\lambda_i) \forall\lambda_i\in\Lambda$ where $\Lambda$ is the set of expected values in the Poisson processes that represent customer behavior for both rental requests and returns. Each location's individual transition probability is $\sim\text{skellam}(\lambda_1,\lambda_2)$ parameterized on the expected returns and expected requests

def transition_probability(next_state, current_state, action):
    total_changes = np.subtract(next_state,current_state)
    customer_changes = np.subtract(total_changes,action)
    # Location indexed array of transition probabilities
    probs = [skellam.pmf(change, mu1=REQUEST_LAM[loc],mu2=RETURN_LAM[loc]) 
             for loc,change in enumerate(customer_changes)]
    # Probability is the product of individual probabilities
    prob = np.prod(probs)
    return prob
  • $r(s,a,s^\prime)$ Reward function

is the product of the cars successfully rented minus the cost of transfers made during action $a$ $$\text{credit}(s^\prime - s - a) - \text{cost}(a)$$

Next, we define an expected return function that will represent both the $$ V(s)\leftarrow \sum_{s^\prime,r} p(s^\prime,r|s,\pi(s)) \left[r+\gamma V(s^\prime)\right] $$ in the case of policy iteration, and $$ V(s)\leftarrow \max_a \sum_{s^\prime,r} p(s^\prime,r|s,a) \left[r+\gamma V(s^\prime)\right] $$ in the case of value iteration.

def expected_return(state, action, value_function):
    returns = 0.0
    # Cost for moving cars
    returns -= MOVE_COST * abs(action)

    # New state after moving cars
    cars = np.minimum(np.add(state,[-action,action]),MAX_CARS)

    state_space = [*cartesian_product(*[range(i+1) for i in MAX_CARS])]

    # Revenue from rentals
    for rental_requests in state_space:
        # Probability of rental requests
        prob = (poisson.pmf(rental_requests[0], REQUEST_LAM[0]) *
                poisson.pmf(rental_requests[1], REQUEST_LAM[1]))
        
        rentals = np.minimum(cars, rental_requests)
        reward = np.sum(rentals) * RENT_CREDIT
        
        # New state after rentals
        unrented_cars = np.subtract(cars, rentals)

        # Probability of returns
        for returned_cars in state_space:
            prob_return = (poisson.pmf(returned_cars[0], RETURN_LAM[0]) *
                            poisson.pmf(returned_cars[1], RETURN_LAM[1]))

            new_state = np.minimum(np.add(unrented_cars,returned_cars),MAX_CARS)
            returns += prob * prob_return *(
                reward + DISCOUNT * value_function[*new_state])

    return returns

def modified_return(state, action, value_function):
    returns = 0.0
    # Cost for moving cars
    returns -= MOVE_COST * abs(action)

    # Employee Shuttling
    if action > 0:
        returns += MOVE_COST

    # New state after moving cars
    cars = np.minimum(np.add(state,[-action,action]),MAX_CARS)

    state_space = [*cartesian_product(*[range(i+1) for i in MAX_CARS])]

    # Revenue from rentals
    for rental_requests in state_space:
        # Probability of rental requests
        prob = (poisson.pmf(rental_requests[0], REQUEST_LAM[0]) *
                poisson.pmf(rental_requests[1], REQUEST_LAM[1]))
        
        rentals = np.minimum(cars, rental_requests)
        reward = np.sum(rentals) * RENT_CREDIT
        
        # New state after rentals
        unrented_cars = np.subtract(cars, rentals)

        # Probability of returns
        for returned_cars in state_space:
            prob_return = (poisson.pmf(returned_cars[0], RETURN_LAM[0]) *
                            poisson.pmf(returned_cars[1], RETURN_LAM[1]))

            new_state = np.minimum(np.add(unrented_cars,returned_cars),MAX_CARS)
            returns += prob * prob_return * (
                reward + DISCOUNT * value_function[*new_state])

            # New Parking Fee (additional $4 fee for each location storing more than 10 cars overnight)
            returns -= sum(new_state>MAX_CARS/2)*4

    return returns

def policy_iteration(policy = None, value_function = None):
    # 1. Initialization
    value_function = value_function or np.zeros(np.add(MAX_CARS,1))
    policy = policy or np.zeros((MAX_CARS[0] + 1, MAX_CARS[1] + 1), dtype=int) 
    # For the arbitrary location version `np.zeros((*np.add(MAX_CARS,1),NUM_LOCATIONS))`
    policy_stable = False

    iteration = 0
    while not policy_stable:
        # 2. Policy Evaluation
        while True:
            delta = 0
            for index, action in np.ndenumerate(policy):
                old_value = value_function[index]
                new_value = expected_return((index), action, value_function)
                value_function[index] = new_value
                delta = max(delta, abs(old_value - new_value))
            if delta < THETA:
                break

        # 3. Policy Improvement
        policy_stable = True
        for index, old_action in np.ndenumerate(policy):
            action_returns = []
            for action in range(-MAX_MOVE, MAX_MOVE + 1):
                if (0 <= index[0] - action <= MAX_CARS[0] 
                and 0 <= index[1] + action <= MAX_CARS[1]):
                    action_returns.append(
                        expected_return((index), action, value_function))
                else:
                    action_returns.append(-float('inf'))
            new_action = np.argmax(action_returns) - MAX_MOVE
            policy[index] = new_action
            if old_action != new_action:
                policy_stable = False

        print("Policy Grid after iteration {}: ".format(iteration + 1))
        plot_policy(policy)
        iteration += 1

    return policy, value_function

(c) Policy Insights¶

Although inefficiencies in the code above and limitations in the hardware employed to run the scripts prevented a full sized running of the solution, the general nature of the change in the policy can be distinctly seen. In particular, Jack is far more likely make the move that occurs free of charge. This resulted in the 'banding' being a bit wider than under the original problem conditions.

The extreme corner values increased to a higher number than in the base problem; but at this scale neither corner meets the maximum move limit. This could mean that the unconstrained optimal at those corners is higher, with the observable result being an increase in the areas of maximum movement (specifically the 5 and -4 on page 81).

Initially, the new lot fee was left to initiate at 10 cars. The effect of this change cannot be seen the smaller maximums. In order to attempt to recapture the probable effect of this factor we redefined this to be half of the location's max capacity.

Once the lot fee incurrence was reduced to actually effect the problem at this level the difference can be seen in the behavior on the upper right hand quadrant. In the original problem there is no consequence to being in this region, in fact having the inventory to service greater demand will be a potential benefit. Once the cost is added, there is a greater incentive to move vehicles to the other location to reduce the probability of having to rent an overflow lot. The new motive force can be visualized as:

def value_iteration(policy = None, value_function = None):
    # 1. Initialization
    value_function = value_function or np.zeros(np.add(MAX_CARS,1))
    policy = policy or np.zeros((MAX_CARS[0] + 1, MAX_CARS[1] + 1), dtype=int) 
    
    iteration = 0
    while True:
        delta = 0
        for index, old_value in np.ndenumerate(value_function):
            action_returns = []
            for action in range(-MAX_MOVE, MAX_MOVE + 1):
                if (0 <= index[0] - action <= MAX_CARS[0] 
                and 0 <= index[1] + action <= MAX_CARS[1]):
                    action_returns.append(
                        expected_return((index), action, value_function))
                else:
                    action_returns.append(-float('inf'))
            
            value_function[index] = max(action_returns)
            delta = max(delta, abs(old_value - value_function[index]))
        
        print(f"Value function update iteration {iteration+1}, delta: {delta}")
        if delta < THETA:
            break
        
        iteration += 1

                    expected_return((index), action, value_function))
            else:
                action_returns.append(-float('inf'))

(e) Algorithm Comparison¶

It is possible that that all observations are the result of bias in the coding of each algorithm.

Value iteration seemed to be a more straightforward implementation than policy iteration. However, when the two were implemented with the intent to keep their operation as similar as possible the policy iteration concluded in significantly shorter time. The resulting policies were quite similar, but under the extended problem the behavior in the quadrant marking overflow lot usage appears to blend more with the behavior outside the quadrant at the edges.
